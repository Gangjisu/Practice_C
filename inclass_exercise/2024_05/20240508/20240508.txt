extern
    int gVar;
    extern int gVar; extern을 사용하지 않아도 사용할 수 있다. 여러개를 사용해도 오류가 나지는 않는다.
    
    참조의 정의 차이점
    int gVar; 참조.
    int gVar = 0; 정의 초기화를 해줌

    int gVar = 2; 이렇게하면 재정의 에러가 발생
    extern int gVar = 2; 확인할 수 없는 외부 참조라는 런타임/실행시간 에러가 발생할 수 있다. 
    
    전역 변수 정의 파일
    인플리테이션 파일 .c
    정의 파일 안에서는 extern을 사용하지 않아도 된다.
    함수의 프로토타입 또는 .h파일에 두는 거조심

    전역변수 접근 범위 제한이 있기에
    정역변수와 정적변수를 적절하게 사용하여 보완하자.

    함수는 기본적으로 전역 함수이다.
    앞에 변수 마찬가지로 extern을 써도 되고 안써도 된다.

    자동 초기화 전역 변수 선언문의 메모리 할당양과 정의한 전역 변수 선언문의 메모리 할당양은 다르다.

    -힙메모리 세그먼트 메모리와 같은 메모리의 종류 알기

    getch() 함수는 표준이 아니지만, _gectch()는 표준이다.

    우리가 사용하는 int i; 는 엄밀히 말하면 변수 i를 정의한 것이다. 
    선언 만으로는 프로그램에 영향을 주지 않는 것을 말한다. 
    예를 보자면 선언은 틀이다. 정의는 틀로 찍어낸 것 실물이다.
